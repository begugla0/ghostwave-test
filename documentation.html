GhostWave

GhostWave — система управления VPN-инфраструктурой на базе протокола GhostNet. Трафик полностью неотличим от обычного HTTPS: используется точный TLS-fingerprint браузера Chrome, адаптивный traffic shaping и полная эмуляция реального веб-сайта для активного зондирования.
🖥️
Panel
Центральный сервер управления. Пользователи, ноды, трафик, Telegram бот. Один на всю инфраструктуру.
⚡
Node
Агент на каждом VPN-сервере. Запускает GhostNet daemon, репортует трафик, получает список разрешённых пользователей.
📱
Client
Кроссплатформенный клиент для Windows / Linux / macOS. Подключается через subscription URL.
🤖
Telegram Bot
Управление доступом: администратор создаёт пользователей, пользователи получают подписку прямо в боте.

04 — Единый установщик install.sh

Вся установка — Panel, Node или оба сразу — выполняется одним скриптом. Он сам проверяет систему, устанавливает Docker, проверяет DNS, генерирует ключи, настраивает файрвол и запускает сервисы.
🖥️
Режим Panel
Управляющий сервер: API, БД, Caddy/SSL, Telegram бот. Один на всю инфраструктуру.
⚡
Режим Node
VPN-сервер: GhostNet daemon + агент. Устанавливается на каждый отдельный сервер.
🔧
Panel + Node
Всё на одном сервере. Удобно для тестирования или небольших инсталляций.

Что делает установщик автоматически

ШагДействие
Проверка системыRoot, ОС, RAM, диск, интернет, Python3, внешний IP
ЗависимостиDocker 24+, Docker Compose v2, iproute2/iptables (для Node)
Ввод данныхДомен, логин/пароль, Telegram, порты — интерактивно с подсказками
Проверка DNSdig A-запись → сверяет с IP сервера → предлагает подождать propagation
Проверка TelegramЗапрос к Telegram API /getMe → показывает @username бота
Генерация ключейDB_PASSWORD, JWT_SECRET, SECRET_KEY, NODE_API_KEY, REDIS_PASSWORD, GhostNet secret
Файрволufw: SSH + нужные порты; порт агента ограничивается только IP Panel
Конфиги.env (600), Caddyfile, docker-compose.yml, /etc/ghostnet/config.json
ЗапускDocker pull → postgres/redis healthcheck → panel → caddy → node agent
Финальная проверкаcurl /health локально и через домен, статус всех контейнеров
Отчёт/root/ghostwave-install-ДАТА.txt с паролями (права 600)
⚠️ Требования перед запуском: Ubuntu 20.04+ или Debian 11+, запуск от root, открытый доступ в интернет. DNS A-запись на домен Panel должна уже существовать — установщик её проверит.

07 — Режим Panel + Node (один сервер)

Используется для тестирования или небольших инсталляций (1 сервер = всё). Выберите режим 3 в установщике.

sudo bash install.sh

  Ваш выбор [1]: 3

Установщик последовательно запросит данные для Panel (домен, Telegram, пароль) и для Node (домен маскировки, порты). Все ключи генерируются автоматически, NODE_API_KEY передаётся между компонентами без ручного копирования.

⚠️ Режим Panel + Node на одном сервере не рекомендуется для production-использования: если сервер заблокируют, недоступны будут и панель управления, и VPN. Используйте отдельные серверы для Panel и каждой Node.

10 — Управление пользователями

Создание пользователя

curl -X POST https://panel.yourdomain.com/api/users \
  -H "Authorization: Bearer $TOKEN" \
  -H "Content-Type: application/json" \
  -d '{
    "username": "ivan",
    "traffic_limit": 10737418240,
    "expires_at": "2025-12-31T23:59:59Z",
    "note": "Иван Иванов",
    "node_ids": [1, 2]

Параметры:

ПолеТипОписание
usernamestringЛатиница, цифры, _ - . (уникальный)
traffic_limitintБайты. 0 = безлимит. 10 ГБ = 10737418240
expires_atISO datetimeДата окончания. null = бессрочно
node_idsarrayID разрешённых нод. [] = все ноды
telegram_idintTelegram ID для привязки к боту

Управление пользователем

# Просмотр
curl https://panel.yourdomain.com/api/users/1 \
  -H "Authorization: Bearer $TOKEN"

# Сброс трафика
curl -X POST https://panel.yourdomain.com/api/users/1/reset-traffic \
  -H "Authorization: Bearer $TOKEN"

# Блокировка
curl -X PATCH https://panel.yourdomain.com/api/users/1 \
  -H "Authorization: Bearer $TOKEN" \
  -H "Content-Type: application/json" \
  -d '{"status": "disabled"}'

# Увеличить лимит
curl -X PATCH https://panel.yourdomain.com/api/users/1 \
  -H "Authorization: Bearer $TOKEN" \
  -H "Content-Type: application/json" \
  -d '{"traffic_limit": 21474836480}'

# Удаление
curl -X DELETE https://panel.yourdomain.com/api/users/1 \
  -H "Authorization: Bearer $TOKEN"

12 — Subscription URL

Каждый пользователь получает уникальный URL вида:

https://panel.yourdomain.com/sub/TOKEN
⛔ Важно: GhostNet — полностью собственный протокол. Он несовместим с v2rayN, NekoBox, Clash, sing-box и другими стандартными клиентами. Эти приложения знают только встроенные протоколы (VLESS, VMess, Trojan, Shadowsocks и др.) и не умеют работать с произвольным кастомным протоколом. Для подключения используйте только GhostWave Client.

Форматы конфигурации

URLФорматДля
/sub/TOKEN?fmt=json JSON (нативный) GhostWave Client — единственный поддерживаемый клиент
/sub/TOKEN?fmt=uri ghostnet://base64 GhostWave Client — импорт по URI
/sub/TOKEN/info JSON Публичная информация о подписке (без секретов, для любого HTTP-клиента)
ℹ️ Форматы ?fmt=clash и ?fmt=singbox в коде панели существуют как заготовки на будущее — для использования после того, как будут написаны форки этих клиентов с поддержкой GhostNet.

Заголовок Subscription-Userinfo

Все ответы содержат стандартный заголовок для совместимых клиентов:

Subscription-Userinfo: upload=1073741824; download=5368709120; total=10737418240; expire=1735689599

16 — Поддерживаемые клиенты и дорожная карта

⛔ GhostNet — полностью собственный протокол. Стандартные клиенты (v2rayN, NekoBox, Clash, sing-box, Hiddify и др.) не могут с ним работать — они поддерживают только встроенные протоколы (VLESS, VMess, Trojan, Shadowsocks, WireGuard, Hysteria2). Никакого плагина для них не существует.

Сейчас поддерживается

🖥️
GhostWave Client
Windows / Linux / macOS
Python 3.10+ + tkinter GUI.
Единственный рабочий клиент на данный момент.

Дорожная карта клиентов

ПлатформаСтатусЧто нужно
Windows / Linux / macOS ✅ Готово Python + tkinter (текущий клиент)
Android 🚧 В планах Нативное приложение на Kotlin, Android VpnService API
iOS 🚧 В планах Нативное приложение на Swift, NetworkExtension API
Форк NekoBox/sing-box 💡 Идея Добавить GhostNet как новый тип в исходный код — большой объём работы
ℹ️ Это обратная сторона собственного протокола: полная защита от DPI требует написания клиентов с нуля под каждую платформу. Готовые решения (Xray, sing-box) могут использоваться большинством существующих приложений именно потому, что реализуют стандартные протоколы.

17 — Авторизация API

Все API эндпоинты кроме /health и /sub/* требуют JWT токен.

username=admin&password=ВашПароль

# Ответ:
{"access_token": "eyJhbGci...", "token_type": "bearer"}

# Использование:
Authorization: Bearer eyJhbGci...
ℹ️ Токен действует 24 часа по умолчанию (настраивается через JWT_EXPIRE_MINUTES).

19 — Устранение ошибок

Установщик прерывается с ошибкой

# Детали последней ошибки:
cat /tmp/gw_install_last.log

# Чаще всего: нет доступа к интернету, нет прав root, или занят порт
# Запустить повторно — установщик продолжит с того же шага:
sudo bash install.sh

Panel не запускается

# Статус всех контейнеров
docker ps -a

# Логи Panel
docker logs ghostwave-panel --tail 50

# Частая причина 1: БД ещё инициализируется
docker compose -f /opt/ghostwave/docker/docker-compose.yml restart panel

# Частая причина 2: неверный .env
cat /opt/ghostwave/.env  # проверьте DB_PASSWORD, JWT_SECRET

SSL-сертификат не получен (Caddy)

# Логи Caddy
docker logs ghostwave-caddy --tail 30

# Проверить что порт 80 открыт
ufw status | grep 80

# Проверить DNS вручную
dig +short panel.yourdomain.com
# Должен вернуть IP вашего сервера

# Перезапустить Caddy после исправления DNS
docker restart ghostwave-caddy

Нода показывает offline

# 1. Проверить доступность агента с Panel-сервера
curl http://NODE_IP:2095/health

# 2. Убедиться что NODE_API_KEY совпадает
grep NODE_API_KEY /opt/ghostwave/.env          # на Panel
grep NODE_API_KEY /opt/ghostwave-node/.env.node # на Node

# 3. Логи Node Agent
docker logs ghostwave-node --tail 50

# 4. Проверить файрвол на Node
ufw status

# 5. Проверить что Panel-сервер не заблокирован на Node
curl -sf https://panel.yourdomain.com/health

Subscription URL возвращает 403

# Проверить статус пользователя
TOKEN="ваш_токен"
curl https://panel.yourdomain.com/sub/SUB_TOKEN/info

# Продлить подписку
curl -X PATCH https://panel.yourdomain.com/api/users/ID \
  -H "Authorization: Bearer $TOKEN" \
  -H "Content-Type: application/json" \
  -d '{"status":"active","expires_at":"2027-01-01T00:00:00Z"}'

# Сбросить трафик
curl -X POST https://panel.yourdomain.com/api/users/ID/reset-traffic \
  -H "Authorization: Bearer $TOKEN"

Сбросить пароль администратора

# Вариант 1: через psql (если помните старый пароль — необязателен)
docker exec -it ghostwave-postgres psql -U ghostwave -c \
  "UPDATE admins SET hashed_password = '\$2b\$12\$...' WHERE username='admin';"

# Вариант 2: временно изменить в .env и перезапустить
# (Panel создаст нового admin при старте если его нет)
nano /opt/ghostwave/.env
# ADMIN_PASSWORD=НовыйПароль

docker compose -f /opt/ghostwave/docker/docker-compose.yml restart panel

Добавить новую ноду к уже работающей Panel

# На новом VPN-сервере — запустить тот же установщик, режим 2) Node
sudo bash install.sh

# Ввести URL существующей Panel и NODE_API_KEY:
grep NODE_API_KEY /opt/ghostwave/.env  # скопировать с Panel-сервера

Просмотр отчёта установщика (пароли)

ls /root/ghostwave-install-*.txt
cat /root/ghostwave-install-20250115-143022.txt

Бэкап и восстановление БД

# Бэкап
docker exec ghostwave-postgres pg_dump -U ghostwave ghostwave \
  | gzip > /root/ghostwave_backup_$(date +%Y%m%d).sql.gz

# Восстановление
gunzip < /root/ghostwave_backup_20250115.sql.gz \
  | docker exec -i ghostwave-postgres psql -U ghostwave ghostwave

Полный сброс и переустановка

docker compose -f /opt/ghostwave/docker/docker-compose.yml down -v
rm -rf /opt/ghostwave
# Затем запустить установщик заново:
sudo bash install.sh
