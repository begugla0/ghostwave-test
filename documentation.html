GhostWave

GhostWave — система управления VPN-инфраструктурой на базе протокола GhostNet. Трафик полностью неотличим от обычного HTTPS: используется точный TLS-fingerprint браузера Chrome, адаптивный traffic shaping и полная эмуляция реального веб-сайта для активного зондирования.
🖥️
Panel
Центральный сервер управления. Пользователи, ноды, трафик, Telegram бот. Один на всю инфраструктуру.
⚡
Node
Агент на каждом VPN-сервере. Запускает GhostNet daemon, репортует трафик, получает список разрешённых пользователей.
📱
Client
Кроссплатформенный клиент для Windows / Linux / macOS. Подключается через subscription URL.
🤖
Telegram Bot
Управление доступом: администратор создаёт пользователей, пользователи получают подписку прямо в боте.

04 — Подготовка сервера Panel

1
Обновление системы
apt update && apt upgrade -y
apt install -y curl wget git ufw
2
Настройка файрвола
ufw allow 22/tcp    # SSH
ufw allow 80/tcp    # HTTP (редирект на HTTPS)
ufw allow 443/tcp   # HTTPS (Panel)
ufw enable
3
Установка Docker
curl -fsSL https://get.docker.com | sh
systemctl enable docker
systemctl start docker

# Проверка
docker --version
docker compose version
4
DNS-запись

В панели вашего DNS-провайдера добавьте A-запись:

panel.yourdomain.com  →  A  →  IP_вашего_сервера
ℹ️Дождитесь propagation DNS (обычно 1–5 минут). Проверить: ping panel.yourdomain.com

06 — SSL и домен (Caddy)

GhostWave использует Caddy как reverse proxy — он автоматически получает Let's Encrypt сертификат.

1
Редактирование Caddyfile
nano /opt/ghostwave/docker/Caddyfile
# Замените panel.example.com на ваш домен
panel.yourdomain.com {
    reverse_proxy /api/*  ghostwave-panel:3000
    reverse_proxy /sub/*  ghostwave-panel:3000
    reverse_proxy /health ghostwave-panel:3000
    reverse_proxy /*      ghostwave-panel:3000
}
2
Перезапуск Caddy
cd /opt/ghostwave/docker
docker compose restart caddy

# Проверка SSL
curl https://panel.yourdomain.com/health
⚠️ Порт 80 должен быть открыт — Caddy использует его для ACME challenge при получении сертификата.

09 — Установка Node (VPN-сервер)

ℹ️ Выполняется на каждом VPN-сервере отдельно. Для каждого нового сервера — отдельный NODE_ID.
1
Подготовка сервера Node
# Обновление
apt update && apt upgrade -y
apt install -y curl docker.io

# Файрвол
ufw allow 22/tcp
ufw allow 443/tcp   # GhostNet (VPN трафик клиентов)
ufw allow 2095/tcp  # Node Agent API (только от Panel!)
ufw enable
⚠️ В идеале порт 2095 открыть только для IP вашего Panel-сервера:
ufw allow from PANEL_IP to any port 2095
2
Создать запись Node в панели
# Выполняется на вашем компьютере (не на сервере)
# Сначала узнайте NODE_API_KEY из .env Panel

TOKEN="ваш_JWT_токен"

curl -X POST https://panel.yourdomain.com/api/nodes \
  -H "Authorization: Bearer $TOKEN" \
  -H "Content-Type: application/json" \
  -d '{
    "name": "🇩🇪 Germany #1",
    "address": "IP_нового_сервера",
    "port": 2095,
    "country": "DE",
    "city": "Frankfurt",
    "flag_emoji": "🇩🇪",
    "ghostnet_port": 443,
    "ghostnet_domain": "news.example.com",
    "ghostnet_secret": "сгенерируйте_python3_secrets_token_hex32"
  }'

# Сохраните "id" из ответа — это ваш NODE_ID
3
Установка Node Agent
mkdir -p /opt/ghostwave-node
cd /opt/ghostwave-node

# Копируем файлы node/ из репозитория
git clone https://github.com/yourorg/ghostwave.git /tmp/gw
cp -r /tmp/gw/node/* .
cp /tmp/gw/docker/docker-compose.node.yml .
cp /tmp/gw/docker/Dockerfile.node .

# Создаём .env.node
cat > .env.node << EOF
NODE_ID=1                          # ID из предыдущего шага
NODE_API_KEY=ключ_из_env_Panel     # NODE_API_KEY из .env Panel
PANEL_URL=https://panel.yourdomain.com
AGENT_PORT=2095
GHOSTNET_PORT=443
GHOSTNET_DOMAIN=news.example.com   # домен для маскировки
GHOSTNET_SECRET=тот_же_что_в_API   # ghostnet_secret из создания ноды
EOF
4
Запуск Node Agent
docker compose -f docker-compose.node.yml up -d

# Проверка
docker logs ghostwave-node -f

В логах должны появиться строки:

[INFO] GhostWave Node Agent starting (node_id=1)
[INFO] Node Agent ready.
[INFO] Heartbeat sent, 0 active users

Создание пользователя

    "expires_at": "2025-12-31T23:59:59Z",
    "note": "Иван Иванов",

Параметры:

ПолеТипОписание
usernamestringЛатиница, цифры, _ - . (уникальный)
traffic_limitintБайты. 0 = безлимит. 10 ГБ = 10737418240
expires_atISO datetimeДата окончания. null = бессрочно
node_idsarrayID разрешённых нод. [] = все ноды
telegram_idintTelegram ID для привязки к боту

Управление пользователем

# Просмотр
curl https://panel.yourdomain.com/api/users/1 \
  -H "Authorization: Bearer $TOKEN"

# Сброс трафика
curl -X POST https://panel.yourdomain.com/api/users/1/reset-traffic \
  -H "Authorization: Bearer $TOKEN"

# Блокировка
curl -X PATCH https://panel.yourdomain.com/api/users/1 \
  -H "Authorization: Bearer $TOKEN" \
  -H "Content-Type: application/json" \
  -d '{"status": "disabled"}'

# Увеличить лимит
curl -X PATCH https://panel.yourdomain.com/api/users/1 \
  -H "Authorization: Bearer $TOKEN" \
  -H "Content-Type: application/json" \
  -d '{"traffic_limit": 21474836480}'

# Удаление
curl -X DELETE https://panel.yourdomain.com/api/users/1 \
  -H "Authorization: Bearer $TOKEN"

14 — Subscription URL

Каждый пользователь получает уникальный URL вида:

https://panel.yourdomain.com/sub/TOKEN
⛔ Важно: GhostNet — полностью собственный протокол. Он несовместим с v2rayN, NekoBox, Clash, sing-box и другими стандартными клиентами. Эти приложения знают только встроенные протоколы (VLESS, VMess, Trojan, Shadowsocks и др.) и не умеют работать с произвольным кастомным протоколом. Для подключения используйте только GhostWave Client.

Форматы конфигурации

URLФорматДля
/sub/TOKEN?fmt=json JSON (нативный) GhostWave Client — единственный поддерживаемый клиент
/sub/TOKEN?fmt=uri ghostnet://base64 GhostWave Client — импорт по URI
/sub/TOKEN/info JSON Публичная информация о подписке (без секретов, для любого HTTP-клиента)
ℹ️ Форматы ?fmt=clash и ?fmt=singbox в коде панели существуют как заготовки на будущее — для использования после того, как будут написаны форки этих клиентов с поддержкой GhostNet.

Заголовок Subscription-Userinfo

Все ответы содержат стандартный заголовок для совместимых клиентов:

Subscription-Userinfo: upload=1073741824; download=5368709120; total=10737418240; expire=1735689599

18 — Поддерживаемые клиенты и дорожная карта

⛔ GhostNet — полностью собственный протокол. Стандартные клиенты (v2rayN, NekoBox, Clash, sing-box, Hiddify и др.) не могут с ним работать — они поддерживают только встроенные протоколы (VLESS, VMess, Trojan, Shadowsocks, WireGuard, Hysteria2). Никакого плагина для них не существует.

Сейчас поддерживается

🖥️
GhostWave Client
Windows / Linux / macOS
Python 3.10+ + tkinter GUI.

Дорожная карта клиентов

ПлатформаСтатусЧто нужно

19 — Авторизация API

Все API эндпоинты кроме /health и /sub/* требуют JWT токен.

# Получение токена
POST /api/auth/login
Content-Type: application/x-www-form-urlencoded

username=admin&password=ВашПароль

# Ответ:
{"access_token": "eyJhbGci...", "token_type": "bearer"}

# Использование:
Authorization: Bearer eyJhbGci...
ℹ️ Токен действует 24 часа по умолчанию (настраивается через JWT_EXPIRE_MINUTES).

21 — Устранение ошибок

Panel не запускается

docker compose logs panel | tail -30

# Частая причина: БД ещё не готова
docker compose restart panel  # подождать 10с после postgres

Нода показывает offline

# 1. Проверить доступность агента с Panel-сервера
curl http://NODE_IP:2095/health

# 2. Проверить что NODE_API_KEY совпадает в обоих .env

# 3. Проверить логи агента
docker logs ghostwave-node -f

# 4. Проверить файрвол на Node (порт 2095 должен быть открыт)
ufw status

Бэкап и восстановление

# Бэкап
docker exec ghostwave-postgres pg_dump -U ghostwave ghostwave \
  | gzip > ghostwave_backup_$(date +%Y%m%d).sql.gz

# Восстановление
gunzip < ghostwave_backup_20250115.sql.gz \
  | docker exec -i ghostwave-postgres psql -U ghostwave ghostwave
